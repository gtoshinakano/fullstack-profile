## 1. Page Loading State Machine

Controls whether the hero and content render or stay hidden. Driven externally by Pace.js.

```mermaid
stateDiagram-v2
    [*] --> Loading : gabriel.tsx mounts\nloading=true, windowProps={}

    Loading --> Loaded : Pace.js fires 'done' event\nwindowProps set from window.innerWidth/Height

    state Loaded {
        [*] --> WideScreen : width > height
        [*] --> MobileScreen : width <= height
        WideScreen --> MobileScreen : window resize\nwidth becomes <= height
        MobileScreen --> WideScreen : window resize\nwidth becomes > height
    }
```

**States:**
| State | `loading` | `windowProps` | Hero visible? |
|-------|-----------|---------------|--------------|
| Loading | `true` | `{}` | No (hidden) |
| Loaded/WideScreen | `false` | `{ width, height }`, width > height | Yes — WideScreen |
| Loaded/MobileScreen | `false` | `{ width, height }`, width ≤ height | Yes — Mobile |

**Trigger:** `Pace.on('done', callback)` — external CDN library event. 🟢

## 2. Tab Selection State Machine (`hero-dark`)

Controls which panel is displayed in the profile section.

```mermaid
stateDiagram-v2
    [*] --> job : Initial state (default)

    job --> Transitioning_to_projects : onSelect('projects')
    job --> Transitioning_to_partner : onSelect('partner')

    projects --> Transitioning_to_job : onSelect('job')
    projects --> Transitioning_to_partner : onSelect('partner')

    partner --> Transitioning_to_job : onSelect('job')
    partner --> Transitioning_to_projects : onSelect('projects')

    Transitioning_to_job --> job : GSAP animation complete\n+ GA event fired
    Transitioning_to_projects --> projects : GSAP animation complete\n+ GA event fired
    Transitioning_to_partner --> partner : GSAP animation complete\n+ GA event fired

    note right of Transitioning_to_job
        loading = true during transition
        Buttons disabled
        Cannot interrupt
    end note
```

**States:**
| State | `selected` | `loading` | Panel shown |
|-------|-----------|-----------|-------------|
| `job` | `'job'` | `false` | Jobs (work history timeline) |
| `projects` | `'projects'` | `false` | Projects (accordion) |
| `partner` | `'partner'` | `false` | FuturePartner (recruiter message) |
| `Transitioning_*` | previous value | `true` | Previous panel (fading out) |

**Transition duration:** GSAP `stagger: 0.05 × n elements` + `scrollTo` animation 🟢

## 3. Project Accordion State (`hero-dark/Projects`)

Controls which projects show their full detail.

```mermaid
stateDiagram-v2
    [*] --> AllCollapsed : Initial state\nexpanded = []

    AllCollapsed --> PartiallyExpanded : toggleByLabel(label)
    PartiallyExpanded --> PartiallyExpanded : toggleByLabel(label)\n[other label]
    PartiallyExpanded --> AllCollapsed : toggleByLabel(label)\n[last expanded item]
    PartiallyExpanded --> AllExpanded : toggleAll()\nwhen allExpanded=false
    AllExpanded --> AllCollapsed : toggleAll()
    AllCollapsed --> AllExpanded : toggleAll()
```

**Note:** Due to duplicate `label = "requirement"` in data, toggling any "requirement" project transitions 5 projects simultaneously. 🔴

**Transition animation:** Tailwind CSS `max-h-0` → `max-h-screen` (CSS transition, no JS animation) 🟢

## 4. Profile Photo Carousel (scroll-driven, not stateful)

This is not a React state machine but a GSAP scroll-driven sequence. Documented here for completeness.

```mermaid
stateDiagram-v2
    [*] --> Photo1Visible : ScrollTrigger initializes\n#profile-photo-01 visible

    Photo1Visible --> Photo2Visible : User scrolls\nphoto-01 fades out\nphoto-02 fades in

    Photo2Visible --> Photo3Visible : User scrolls more\nphoto-02 fades out\nphoto-03 fades in

    Photo3Visible --> Photo2Visible : User scrolls up
    Photo2Visible --> Photo1Visible : User scrolls up
```

**Photos sequence:**
1. `gabriel-photo.png` — professional photo
2. `gabriel-github.png` — GitHub avatar
3. `gabriel-photo.jpg` — alternate professional photo

**Trigger:** GSAP ScrollTrigger on `#profile-photo` element, `scrub:1`, `start:'top center'`, `end:'10% 10%'` 🟢

## 5. Education Visibility Toggle

Simple two-state toggle within the Jobs component.

```mermaid
stateDiagram-v2
    [*] --> Shown : hasEducation = true (default)
    Shown --> Hidden : User clicks "Hide Degrees"
    Hidden --> Shown : User clicks "Show Degrees"
```

**Effect:** Hides/shows the UNINOVE (2008–2010) and FATEC (2010–2013) education timeline entries. 🟢
