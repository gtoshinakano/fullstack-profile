## Interface

| Símbolo | Assinatura | Retorno | Observação |
|---------|-----------|---------|------------|
| `Analytics` | `()` | `JSX.Element` | Gate de produção; renderiza `<GA />` ou fragment vazio |
| `GAScript` | `()` | `JSX.Element` | Injeta scripts gtag.js e registra listener de rota |
| `ga.pageview` | `(url: string) => void` | `void` | Envia hit de pageview ao GA4 |
| `ga.event` | `({ action, params }) => void` | `void` | Envia evento customizado ao GA4 |
| `ga.hasGa` | `() => void` | — | Guard defensivo: verifica `window && window.gtag` |

**API pública de `ga.ts`** (usada por outras features): 🟢

```ts
// src/lib/ga.ts
interface IEvent {
  action: string
  params: {
    event_category?: string
    event_label?: string
  }
}

export const pageview = (url: string): void => {
  if (window && window.gtag)
    window.gtag('config', process.env.NEXT_PUBLIC_GOOGLE_ANALYTICS, {
      page_path: url,
    })
}

export const event = ({ action, params }: IEvent): void => {
  if (window && window.gtag) window.gtag('event', action, params)
}

export const hasGa = (): void => window && window.gtag
```

> ⚠️ **Reclassificação [Reviewer]:** A variável de ambiente é `NEXT_PUBLIC_GOOGLE_ANALYTICS`, não `NEXT_PUBLIC_GA_ID` como documentado anteriormente. Não existe `GA_TRACKING_ID` exportado. Corrigido com base em `src/lib/ga.ts` confirmado.

## Fluxo Principal — Inicialização

1. `gabriel.tsx` renderiza `<Analytics />` (sempre, independente de `loading`)
2. `Analytics/index.tsx` verifica `process.env.NODE_ENV`
   - Se `!== 'production'`: renderiza fragment vazio `<></>` — GA não é montado
   - Se `=== 'production'`: renderiza `<GA />` (componente `GAScript`)
3. `GAScript` injeta dois `<Script>` tags via Next.js com **`strategy='lazyOnload'`**: 🟢
   - `https://www.googletagmanager.com/gtag/js?id=${NEXT_PUBLIC_GOOGLE_ANALYTICS}`
   - Script inline de inicialização do `window.dataLayer` e `gtag()`
4. `GAScript` registra listener via `useEffect([router.events])`:
   - `router.events.on('routeChangeComplete', handleRouteChange)`
   - Cleanup: `router.events.off('routeChangeComplete', handleRouteChange)` — **cleanup implementado** 🟢
5. `handleRouteChange(url)` chama `ga.pageview(url)` a cada troca de rota

> ⚠️ **Reclassificações [Reviewer]:** (1) `strategy` é `lazyOnload`, não `afterInteractive`. (2) Cleanup do listener está implementado (🟡→🟢). (3) `Analytics` retorna `<></>` não `null`.

## Fluxo Principal — Evento Customizado

```
Feature X chama ga.event({ action: 'menu_click', category: 'projects' })
  ↓
ga.ts: verifica typeof window.gtag === 'function'
  ↓ (produção)
window.gtag('event', 'menu_click', { event_category: 'projects' })
  ↓
GA4 recebe hit e registra no dashboard
```

```
  ↓ (development — gtag não existe)
retorna sem ação — sem erro, sem log
```

## Fluxos Alternativos

- **Bloqueador de anúncios ativo:** `window.gtag` nunca é definido mesmo em produção → guard `typeof window.gtag !== 'function'` retorna silenciosamente → nenhum evento é enviado → nenhum erro
- **GA_TRACKING_ID não configurado:** script carrega com ID `undefined` → GA4 rejeita silenciosamente → sem erro de runtime, mas sem dados coletados
- **Unmount do componente:** listener `routeChangeComplete` deve ser removido no cleanup do `useEffect` via `router.events.off(...)` 🟡

## Dependências

- **`next/router`** — `useRouter()` para `router.events` (listener de troca de rota)
- **`next/script`** — `<Script strategy="afterInteractive">` para carregamento não-bloqueante do gtag
- **`process.env.NODE_ENV`** — gate de produção
- **`process.env.NEXT_PUBLIC_GA_ID`** — Measurement ID do GA4 (variável pública — prefixo `NEXT_PUBLIC_`)

## Riscos e Lacunas

- ~~🟡 **Cleanup do listener de rota:**~~ **Revisão: cleanup está implementado** 🟢 — `router.events.off('routeChangeComplete', handleRouteChange)` no return do `useEffect` em `GoogleAnalytics.tsx:19-21`.
