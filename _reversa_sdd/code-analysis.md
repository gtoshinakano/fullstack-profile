## Executive Summary

`fullstack-profile` is a **single-page personal portfolio** for Gabriel Toshinori Nakano, a Full-Stack Developer based in Tokyo. The application is built with **Next.js 14 (Pages Router)** and deployed as a **static export to GitHub Pages**. It is composed of 7 distinct features analyzed below.

**Architecture pattern:** component-driven SPA with file-based routing, no API layer, no server-side state, no database. All data is embedded as static JSON.

## Feature 1 — hero-section

**Purpose:** Full-viewport animated mountain landscape that serves as the visual entry point. Responsive: renders `WideScreen` (landscape) or `Mobile` (portrait) based on viewport aspect ratio.

### Primary Files
| File | Role |
|------|------|
| `src/components/views/dev/gabriel/HeroSection/WideScreen.tsx` | Desktop/landscape parallax hero |
| `src/components/views/dev/gabriel/HeroSection/Mobile.tsx` | Mobile/portrait parallax hero |
| `src/components/views/dev/gabriel/HeroSection/HeroSvg.js` | Inline SVG text art (two variants) |
| `src/components/views/dev/gabriel/HeroSection/index.ts` | Barrel export |

### Functions & Components

**`WideScreen({ loading: boolean })`** 🟢
- On `loading === false`: initializes 10+ GSAP element positions, attaches ScrollTrigger timeline scrubbing 600px
- Layer z-order (bottom→top): background → `#big-mnt-img` (z=0) → `#uiux-dev-txt` (z=10) → `#mid-mnt-img` (z=30) → clouds (z=40/50) → `#sm-mnt-back` → `#climb-mnt` → `#sm-mnt-front` → `#cta-btn` (z=50)
- `handleAction()`: scrolls to `main` element + fires GA event `hero_cta` (category: "3-3-3 Principle", label: "widescreen")

**`Mobile({ loading: boolean })`** 🟢
- Same structure as WideScreen with adapted initial positions for portrait viewport
- `handleAction()` fires same GA event with label "mobile screen"

**`UXDevWide()`** / **`UXDevMobile()`** 🟢
- Pure SVG components rendering white-filled path text art: "FULL STACK DEV. | developer.p" (wide) and "FULL. ULL. DEV. | developer.p" (mobile)
- No external dependencies, no props

### Key Algorithms

**GSAP Parallax Pipeline** 🟢
```
ScrollTrigger(end: 600, scrub: 1)
  → 10 simultaneous .to() animations in a timeline
  → Each layer moves at different speeds/directions
  → Creates 3D depth illusion from 2D images
  → CTA button fades to opacity:0 at scroll end
```

**Responsive Decision** 🟢
- Decision point is in parent `gabriel.tsx`, not inside the component
- `isWide = windowProps.width && windowProps.height && windowProps.width > windowProps.height`
- Renders after Pace.js `done` event fires (page fully loaded)

### Business Rules
- Loading state persists until Pace.js fires `done` — hero is hidden (`hidden` class) while loading 🟢
- CTA button smoothly disappears as user scrolls — it is a soft nudge, not a gating mechanism 🟢

### Dependencies
- `gsap` (ScrollTrigger, ScrollToPlugin)
- `next/image` (with custom loader for basePath)
- `@/helpers/prefix` (basePath string)
- `@/lib/ga` (analytics events)

## Feature 2 — hero-dark

**Purpose:** Dark-themed profile section below the hero. Contains a 3-tab menu (Profile, Projects, Message) with animated content panels and a GSAP-driven sticky menu that expands on scroll.

### Primary Files
| File | Role |
|------|------|
| `src/components/views/dev/gabriel/HeroDark/index.tsx` | Container, tab state, profile photo carousel, sticky menu |
| `src/components/views/dev/gabriel/HeroDark/Jobs.tsx` | "Profile" tab — work history timeline + education |
| `src/components/views/dev/gabriel/HeroDark/Projects.tsx` | "Projects" tab — accordion project cards |
| `src/components/views/dev/gabriel/HeroDark/FuturePartner.jsx` | "Message" tab — recruiter-facing text (fully i18n) |

### Functions & Components

**`Herodark({ isWide: boolean | 0 | undefined })`** 🟢
- State: `selected: 'job' | 'projects' | 'partner'` (default: `'job'`)
- State: `loading: boolean` (menu transition lock)
- GSAP ScrollTrigger: profile photo crossfade carousel (3 sequential photos: gabriel-photo.png → gabriel-github.png → gabriel-photo.jpg)
- `onSelect(newSelected: string)`: sets loading, fades out `.should-hide` elements, scrolls to `#profile-photo`, then updates `selected` and fires GA event `menu_click` (category: newSelected)

**`Menu({ setSelected, selected, loading })`** 🟢
- GSAP ScrollTrigger: pins the menu, expands it to full viewport width (`window.innerWidth < 768 ? innerWidth : 850`), scales to 1.1, removes border-radius on scroll
- Three `Button` components (Profile, Projects, Message)

**`Button({ children, isActive, ...rest })`** 🟢
- Active state: gradient background (white→slate-300→gray-400), `cursor-not-allowed`, `disabled={true}`
- Inactive state: cursor-pointer, text-ternary, hover:text-figmaBlue

**`Jobs()`** 🟢
- Reads: `@/data/jobs.json` (5 entries), `@/data/stacks.json`, `@/data/swtools.json`
- State: `hasEducation: boolean` (default: true — shows education entries)
- Renders work history in reverse chronological order (lodash `_.reverse([...data])`)
- Hardcoded education entries inserted at index 3 and at the bottom (FATEC 2010-2013, UNINOVE 2008-2010)
- GA events: none directly in Jobs; menu click tracked in parent `onSelect`

**`Projects()`** 🟢
- Reads: `@/data/toshi-projects.json` (19 entries), `@/data/stacks.json`
- State: `expanded: string[]` — list of project `label` strings currently expanded
- `toggleAll()`: expands all or collapses all
- `toggleByLabel(label: string)`: adds/removes label from `expanded` array using `indexOf` + `splice`
- Renders in reverse chronological order (`_.reverse([...data])`)
- Expand animation via Tailwind `max-h-0` → `max-h-screen` transition

**`FuturePartner()`** 🟡
- Fully i18n via `useTranslation('future-partner')` namespace
- Uses `Trans` component from react-i18next for rich text (HTML elements within translations)
- `elemMapping`: maps `'strong'` → `<strong />`, `'a'` → `<a {...props} />`

### State Machine — Tab Selection
```
Initial: selected='job', loading=false

onSelect(newValue):
  loading = true
  GSAP: fade out .should-hide, scroll to #profile-photo
  on complete:
    selected = newValue
    loading = false
    GA.event('menu_click', { category: newValue })
```

### Business Rules
- Tab change is locked during animation (loading=true prevents rapid clicking) 🟢
- Projects are always displayed newest-first (reverse of JSON array order) 🟢
- Education is shown by default but can be hidden 🟢
- `label` field in projects.json acts as expand/collapse key — **duplicate labels exist** (`"requirement"` appears 5 times) 🔴 **BUG:** toggling one "requirement" project expands all with same label simultaneously

### Dependencies
- `gsap`, `dayjs`, `lodash`, `react-i18next`, `next/image`
- `@/data/jobs.json`, `@/data/stacks.json`, `@/data/swtools.json`, `@/data/toshi-projects.json`

### Functions

**`Analytics()`** 🟢
- Renders `<GA />` only when `process.env.NODE_ENV === 'production'`
- Plausible and SimpleAnalytics integrations present but fully commented out

**`GAScript()`** 🟢
- Next.js `<Script strategy='lazyOnload'>` — non-blocking load
- Subscribes to `router.events.on('routeChangeComplete', handleRouteChange)` on mount
- Unsubscribes on unmount (no memory leak)
- `handleRouteChange(url)`: calls `ga.pageview(url)`

**`pageview(url: string)`** 🟢
- Calls `window.gtag('config', GA_ID, { page_path: url })` — guarded by `window && window.gtag`

**`event({ action, params })`** 🟢
- Calls `window.gtag('event', action, params)` — guarded by `window && window.gtag`

**`hasGa()`** 🟡
- Returns `window && window.gtag` — name implies boolean check but return type is `void` in signature, actual return is the expression value

### GA Events Fired
| Event | Location | Parameters |
|-------|----------|-----------|
| `menu_click` | `HeroDark/index.tsx:onSelect` | `event_category: 'job'|'projects'|'partner'` |
| `hero_cta` | `HeroSection/WideScreen.tsx:handleAction` | `event_category: '3-3-3 Principle'`, `event_label: 'widescreen'` |
| `hero_cta` | `HeroSection/Mobile.tsx:handleAction` | `event_category: '3-3-3 Principle'`, `event_label: 'mobile screen'` |

### Business Rules
- GA is completely disabled in development — no tracking, no console noise 🟢
- GA measurement ID is injected at build-time via env var, not hardcoded in source 🟢
- GA ID: `G-RH74DXLKZL` (set in CI/CD workflow, not in source code) 🟢

## Feature 5 — i18n

**Purpose:** Multi-language support for 3 locales (English, Japanese, Brazilian Portuguese) via route-based locale switching and browser language auto-detection.

### Primary Files
| File | Role |
|------|------|
| `next-i18next.config.js` | Root i18n config — locales list, default locale |
| `src/lib/getStatic.js` | Static path/prop generators for [locale] pages |
| `src/lib/languageDetector.js` | Browser language detection config |
| `src/lib/redirect.js` | Client-side locale redirect logic |
| `public/locales/en/common.json` | EN translations |
| `public/locales/ja/common.json` | JA translations |
| `public/locales/pt-BR/common.json` | PT-BR translations |
| `public/locales/*/future-partner.json` | Recruiter section translations (3 locales) |

### Functions

**`getI18nPaths()`** 🟢
- Returns `[{ params: { locale: 'en' } }, { params: { locale: 'ja' } }, { params: { locale: 'pt-BR' } }]`
- Used as `getStaticPaths` return for `[locale]` pages

**`makeStaticProps(ns)`** 🟢
- HOF returning `getStaticProps` that calls `serverSideTranslations(locale, ns)`
- Default namespaces: `['common', 'future-partner']`

**`useRedirect(to?)`** 🟢
- Detects browser language via `languageDetector.detect()`
- Computes basePath: `/fullstack-profile/` in production, `/` in dev
- Guards against redirect loops: if current route is `/404` and target starts with `basePath + detectedLng`, redirects to `basePath + detectedLng + route`
- Caches detected language via `languageDetector.cache(detectedLng)`
- Calls `router.replace(basePath + detectedLng)` on every render (runs as effect) 🔴 **POTENTIAL BUG:** no dependency array → runs on every render, not just once on mount

**`Redirect`** / **`getRedirect(to)`** 🟢
- Thin wrappers around `useRedirect` for use as page components

### Route Structure
```
/                          → redirects to /<detected-locale>
/<locale>/                 → redirects to /<locale>/dev/gabriel-toshinori-nakano/
/<locale>/dev/gabriel/     → legacy alias, redirects to gabriel-toshinori-nakano
/<locale>/dev/gabriel-toshinori-nakano/  → main portfolio page
```

### Supported Locale Keys (common.json)
See `data-dictionary.md` for full translation key schema.

### Business Rules
- Default locale is `en` — English is the fallback for unknown browser languages 🟢
- Locale is part of the URL path, not a query param or cookie — SEO-friendly 🟢
- Article content (Introduction.tsx) is **not translated** — always renders in English regardless of locale 🟡 INFERRED
- Language switcher (ChangeLanguage) is always visible, fixed top-right 🟢

## Cross-Cutting Concerns

### Loading Sequence
```
1. Browser loads /fullstack-profile (GitHub Pages)
2. Next.js static page loads, client-side JS executes
3. useRedirect() fires → router.replace to /fullstack-profile/<locale>/dev/...
4. gabriel.tsx mounts → loading=true
5. Pace.js CDN script loads → monitors resource loading
6. Pace.js fires 'done' → loading=false, windowProps set
7. isWide computed → HeroSection.WideScreen or Mobile renders
8. GSAP animations initialize
9. GA lazyOnload scripts fire
```
