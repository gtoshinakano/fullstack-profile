## Purpose of the System

This is a **personal developer portfolio** for Gabriel Toshinori Nakano ("Toshi"), a Full-Stack Developer. Its goal is to:

1. Present the developer's professional identity to recruiters and potential partners
2. Showcase a curated timeline of work history and personal projects
3. Deliver the "3-3-3 Principles for a Better UX" — a personal UI/UX guide written by the author
4. Demonstrate the author's own UI/UX skills through the design and implementation of the site itself

The site is the product *and* the portfolio piece simultaneously. 🟢

## Business Rules

### BR-01 — Page renders only after Pace.js reports loading complete 🟢
The hero and all content sections are hidden (`display: none`) while Pace.js monitors page resource loading. Only after Pace fires the `done` event does `loading` become `false` and the UI reveal itself. This prevents users from seeing incomplete/broken parallax states.

**Location:** `pages/[locale]/dev/gabriel-toshinori-nakano/gabriel.tsx`

### BR-02 — Responsive breakpoint is aspect ratio, not viewport width 🟢
Unlike conventional mobile breakpoints (e.g. `< 768px`), the WideScreen/Mobile decision uses `width > height`. A tablet in landscape shows WideScreen; the same tablet in portrait shows Mobile. This was a deliberate choice to optimize for the mountain landscape composition.

**Location:** `gabriel.tsx:37-40`

### BR-03 — Tab changes are locked during animations 🟢
When a tab is changing, `loading = true` disables the menu buttons. This prevents incomplete GSAP animation states from stacking. The transition sequence (fade → scroll → update state) must complete before the next selection is accepted.

**Location:** `HeroDark/index.tsx:50-68`

### BR-04 — Projects always display newest-first 🟢
`toshi-projects.json` is ordered chronologically (oldest first). The Projects component applies `_.reverse([...data])` before render so the most recent work appears at the top. Jobs applies the same pattern.

**Location:** `HeroDark/Projects.tsx:72`, `HeroDark/Jobs.tsx:35`

### BR-05 — Education entries are hardcoded in JSX, not in jobs.json 🟢
The two academic degrees (UNINOVE 2008–2010, FATEC 2010–2013) are embedded directly in `Jobs.tsx` as static JSX. They do not appear in `jobs.json`. This separates professional work history (data-driven) from education (static).

**Location:** `HeroDark/Jobs.tsx:90-113, 195-225`

### BR-06 — Education is visible by default but user-hideable 🟢
`hasEducation` defaults to `true`. The toggle button alternates display of the two education blocks. The most recent education entry is inserted at index 3 in the timeline (after 3 jobs), and the oldest at the bottom.

**Location:** `HeroDark/Jobs.tsx:19, 90-113`

### BR-07 — Duplicate `label` field causes grouped accordion behavior 🟢
The `label` field in `toshi-projects.json` was intended as a unique key (documented in `Projects.tsx:25` comment: `"Unique Key of project"`). However, 5 entries share the label `"requirement"`. When any of these is expanded, all 5 expand simultaneously because the state check uses `expanded.includes(label)`.

**Location:** `src/data/toshi-projects.json` (entries: AXA requirement design, NBWF, local-file-uploader, web-template-string, underwriting improvement)

### BR-08 — Analytics disabled completely in development 🟢
`process.env.NODE_ENV === 'production'` gates the entire Analytics component. No tracking, no errors, no console output in dev. GA events also guard against undefined `window.gtag`.

**Location:** `Analytics/index.tsx:7`, `ga.ts:17-28`

### BR-09 — Language switcher requires prefix for GitHub Pages 🟢
All `router.push()` calls in `ChangeLanguage` prepend `prefix` (the basePath). Without it, language switches on GitHub Pages redirect to the repo root instead of the correct sub-path. This was a production bug fixed on Aug 10, 2022 (commit `e33d3e7`).

**Location:** `ChangeLanguage/index.tsx`, fix commit `e33d3e7`

### BR-10 — Article content (Introduction) is English-only 🟡 INFERRED
`Introduction.tsx` has no `useTranslation` hook and no i18n keys. The article renders in English regardless of the visitor's locale. This is likely a deliberate scope decision (the article is English-language practice for the author, per the article's own text) rather than an oversight.

**Location:** `MainContent/Introduction.tsx`

### BR-11 — Social links at article bottom are decorative placeholders 🟢
The "Let's connect", "Share this page", "Buy me a Coffee" text at the bottom of Introduction.tsx are plain `<span>` elements with no `href`, `onClick`, or any interactive behavior. They appear as visual decoration only.

**Location:** `Introduction.tsx:621`

### BR-12 — `reloadOnPrerender: true` has no effect in static export 🟡 INFERRED
`next-i18next.config.js` includes `reloadOnPrerender: true`. This option is only meaningful in SSR/SSG with a running server. In static export mode (`output: 'export'`), it is silently ignored. It is a remnant of the early SSR-based i18n attempt that was abandoned in commit `63139d7`.

**Location:** `next-i18next.config.js:5`, historical commit `63139d7`

### BR-13 — GitHub Pages deployment is fully automated on push to `main` 🟢
Any push to `main` triggers the GitHub Actions workflow, which builds the Next.js static export and deploys it to the `gh-pages` branch via `JamesIves/github-pages-deploy-action`. No manual deploy step required.

**Location:** `.github/workflows/deploy.yml`

### BR-14 — Technology opinions are server-side only; never exposed to frontend 🟢 **(Feature 006)**
Each stack entry in `stacks.json` includes an `opinion` field (Gabriel's technical perspective). This field is consumed **only** by the Cloudflare Worker's `buildTechPerspective()` function to enrich the AI system prompt. The frontend components (Projects.tsx, Jobs.tsx) never import or access the `opinion` field; it remains invisible in the browser. This separation ensures opinions are AI-context only and never used for visual rendering.

**Location:** `src/data/stacks.json`, `worker/src/systemPrompt.ts:buildTechPerspective()`, verified by grep in `src/components/`

**Validation:** All 39 stacks have non-empty opinion field (confirmed 2026-05-20)
