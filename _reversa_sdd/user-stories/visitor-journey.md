## Jornada 1 — Recrutador no Japão (primeira visita)

```
US-01: Como recrutador japonês,
       quero que o portfólio detecte automaticamente meu idioma
       para não precisar configurar manualmente o locale.

  Cenário principal:
    Dado que meu browser está configurado para japonês (ja-JP)
    Quando acesso a URL raiz do portfólio
    Então sou redirecionado para /ja/dev/gabriel-toshinori-nakano/
    E o conteúdo da aba Profile está em japonês

US-02: Como recrutador japonês,
       quero visualizar o histórico profissional em ordem cronológica reversa
       para avaliar rapidamente a experiência mais recente.

  Cenário principal:
    Dado que estou na aba Profile (Jobs)
    Quando a página carrega
    Então vejo o job mais recente (AXA) no topo da timeline
    E cada entrada mostra empresa, cargo e período

US-03: Como recrutador japonês,
       quero enviar uma mensagem direta ao autor
       para iniciar um processo seletivo.

  Cenário principal:
    Dado que estou interessado no perfil do autor
    Quando clico na aba "Message" (FuturePartner)
    Então vejo o texto de apresentação em japonês
    E há instruções ou informações de contato
```

## Jornada 3 — Par Técnico (avaliação de projetos)

```
US-06: Como desenvolvedor técnico,
       quero ver todos os projetos do autor de uma vez
       para ter uma visão rápida do portfólio completo.

  Cenário principal:
    Dado que estou na aba Projects com todos os itens colapsados
    Quando clico em "Expand All"
    Então todos os 19 projetos expandem seus detalhes simultaneamente

US-07: Como desenvolvedor técnico,
       quero avaliar as tecnologias usadas em cada projeto
       para verificar compatibilidade com minha stack.

  Cenário principal:
    Dado que expandi um projeto
    Quando leio os detalhes
    Então consigo ver o stack técnico (React, Next.js, etc.) e o papel do autor

US-08: Como desenvolvedor técnico,
       quero acessar o repositório GitHub de um projeto
       para analisar o código real.

  Cenário principal:
    Dado que um projeto tem link do GitHub
    Quando clico no link
    Então sou direcionado para o repositório no GitHub

  Cenário alternativo:
    Dado que um projeto não tem link do GitHub
    Quando visualizo os detalhes
    Então o link de GitHub não aparece (sem link quebrado)
```

## Jornada 5 — Autor (manutenção)

```
US-11: Como autor do portfólio,
       quero adicionar um novo job ao meu histórico
       para manter o portfólio atualizado após uma mudança de emprego.

  Critério de aceite:
    Dado que recebi uma nova oferta e comecei um novo emprego
    Quando edito src/data/jobs.json adicionando a nova entrada
    E faço push para a branch main
    Então o GitHub Actions constrói e faz deploy automaticamente
    E o novo job aparece no topo da timeline (mais recente)
    E o job anterior perde o indicador "ongoing"

US-12: Como autor do portfólio,
       quero que o deploy seja automático no push para main
       para não precisar fazer deploy manual.

  Cenário principal:
    Dado que fiz commit e push para main
    Quando o GitHub Actions executa
    Então o workflow build + deploy completa sem erro
    E o site atualizado está disponível em gtoshinakano.github.io/fullstack-profile/
```
