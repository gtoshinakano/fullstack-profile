## Fluxo Principal (WideScreen)

1. **gabriel.tsx monta** — `loading=true`, `windowProps={}` — hero oculto
2. **Pace.js dispara `done`** — callback seta `loading=false` e `windowProps` com `window.innerWidth/Height`
3. **`isWide` calculado** — `width > height` → `true` para landscape
4. **`WideScreen` renderiza** — 10 camadas de imagem empilhadas via CSS absolute/z-index
5. **`useEffect([loading])` executa** — quando `loading=false`, GSAP configura posições iniciais com `gsap.set()` e cria timeline 🟢
6. **GSAP Timeline criada** — `scrub:1`, sem trigger explícito (usa trigger padrão), `end: 600` (pixels absolutos, não `+=600px`) 🟢
7. **Cada camada recebe tweens** — translateY/opacity distintos por camada, gerando profundidade
8. **Elementos textuais animados** — título, subtítulo e CTA aparecem com fade/slide escalonado
9. **Scroll do usuário ativa scrub** — GSAP move as camadas proporcionalmente ao scroll position
10. **CTA clicado** → `gsap.to(window, { scrollTo: 'main', duration: 0.5 })` 🟢

## Fluxo Principal (Mobile)

1. **gabriel.tsx monta** → `loading=false`, `isWide=false`
2. **`Mobile` renderiza** — imagem de fundo estática (PNG) via `<Image loader={customLoader}>`
3. **`HeroSvg` renderizado** sobre a imagem — posição absoluta, z-index superior
4. **CTA button renderizado** — mesmo comportamento de scroll que WideScreen
5. **Sem GSAP** — nenhuma animação em Mobile

## Fluxos Alternativos

- **Resize de landscape para portrait:** `window.resize` listener em `gabriel.tsx` atualiza `windowProps` → `isWide` torna-se `false` → `Mobile` substitui `WideScreen`. GSAP ScrollTrigger do WideScreen não é destruído explicitamente — possível memory leak. 🟡
- **Pace.js CDN offline:** `Pace.on('done', cb)` nunca é chamado → `loading` permanece `true` → hero nunca aparece. Sem fallback implementado. 🔴
- **JavaScript desabilitado:** Next.js gera HTML estático mas todas as animações e o gate do Pace.js requerem JS. A página renderiza em branco. 🟡

## Dependências

- **GSAP 3.12** (`gsap`, `ScrollTrigger`, `ScrollToPlugin`) — biblioteca de animação; todas as animações WideScreen dependem dela
- **Pace.js (CDN)** — gate de loading; injetado via `<script>` em `layout/Public/index.tsx`
- **`prefix.ts`** — todas as URLs de imagem passam pelo helper de basePath
- **`customLoader.ts`** — loader identity function para todos os `<Image>` do hero
- **`gabriel.tsx`** — controla `loading` e `windowProps`; `HeroSection` não acessa `window` diretamente

## Decisões de Design Identificadas

| Decisão | Evidência no código | Confiança |
|---------|---------------------|-----------|
| Breakpoint por aspect ratio (`width > height`) em vez de viewport width px | `gabriel.tsx:37-40` | 🟢 |
| 10 camadas PNG empilhadas via CSS absolute em vez de canvas/WebGL | `WideScreen.tsx` — 10 `<Image>` com z-index sequencial | 🟢 |
| `scrub:1` em vez de duração fixa — animação sincronizada ao scroll | `WideScreen.tsx` — GSAP timeline com `scrub:1` | 🟢 |
| Gate de render via Pace.js `done` event em vez de `onLoad` do browser | `gabriel.tsx` — `Pace.on('done', ...)` | 🟢 |
| `useEffect([loading])` para registrar GSAP (executa após render, quando loading=false) | `WideScreen.tsx:17-107` | 🟢 |
| `customLoader` identity function — sem otimização de imagem | `src/helpers/customLoader.ts`, `next.config.js` (ADR-004) | 🟢 |

## Estado Interno

`HeroSection`, `WideScreen` e `Mobile` são **stateless**. Todo estado é gerenciado no pai `gabriel.tsx`:

| Campo | Tipo | Valor inicial | Quando muda |
|-------|------|---------------|-------------|
| `loading` | `boolean` | `true` | `false` após Pace.js `done` |
| `windowProps` | `{ width: number, height: number }` | `{}` | Após Pace.js `done` + em cada `resize` |

GSAP mantém estado interno de animação via `ScrollTrigger.getAll()` — não exposto como React state.

## Riscos e Lacunas

- ~~🔴 Sem fallback para Pace.js offline~~ → **Resolvido:** timeout de 5 segundos em `gabriel.tsx` que força `loading=false` (✅ Respondida — Pergunta 1) 🟢
- 🟡 **ScrollTrigger não é destruído no resize** — ao alternar WideScreen → Mobile, a instância GSAP pode continuar ativa. Sem evidência de `ScrollTrigger.kill()` no cleanup do `useEffect`/`useLayoutEffect`.
- 🟡 **Imagens sem dimensões fixas** — `<Image>` com `customLoader` não beneficia de lazy loading otimizado; em conexões lentas, as 10 camadas PNG podem atrasar o render mesmo após o Pace gate.
