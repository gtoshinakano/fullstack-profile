# i18n

> Generated by Reversa Writer · 2026-05-17

## Visão Geral

A feature i18n entrega suporte a três idiomas (EN, JA, PT-BR) em um site estático exportado pelo Next.js. Como o Next.js built-in i18n requer servidor para detecção de locale e é incompatível com `output: 'export'`, toda a lógica foi reimplementada no lado do cliente: detecção de idioma do browser, redirect automático para a rota correta e carregamento de traduções em tempo de build. O resultado é um portfólio multilíngue sem servidor, com URL limpa por locale (`/en/`, `/ja/`, `/pt-BR/`). (Ver ADR-002 para o histórico da decisão.)

## Responsabilidades

- Detectar o locale preferido do browser na primeira visita e redirecionar para a rota correta 🟢
- Servir o conteúdo na rota `/{locale}/dev/gabriel-toshinori-nakano/` 🟢
- Carregar as strings de tradução corretas para o locale ativo via `useTranslation` 🟢
- Proteger contra loop de redirect na rota `/404` 🟢
- Permitir que o visitante troque o idioma manualmente via widget `ChangeLanguage` 🟢
- Garantir que trocas de idioma via widget incluam o `prefix` (basePath do GitHub Pages) 🟢

## Regras de Negócio

- **BR-09** — Todas as chamadas `router.push()` para troca de locale devem incluir o `prefix` (basePath `/fullstack-profile`). Sem o prefix, a troca de idioma redireciona para a raiz do repositório em vez da rota correta. 🟢
- **BR-12** — `reloadOnPrerender: true` em `next-i18next.config.js` não tem efeito em static export. É um resquício da implementação SSR abandonada e pode ser removido sem consequências. 🟡
- Locales suportados: `en` (padrão), `ja`, `pt-BR`. Qualquer locale não reconhecido pelo `languageDetector` faz fallback para `en`. 🟢
- `serverSideTranslations` é chamado em `getStaticProps` de cada página — funciona em tempo de build, não em runtime. O nome é enganoso no contexto de static export. 🟢
- O hook `useRedirect` roda em `pages/index.ts` (e possivelmente em outras rotas raiz) — ao detectar `/404` na rota atual, a lógica de redirect é suspensa para evitar loop. 🟢
- O artigo `Introduction.tsx` não é traduzido — permanece em inglês independentemente do locale. 🟢 (BR-10)

## Critérios de Aceitação

```gherkin
Dado que um visitante com browser configurado para PT-BR acessa a URL raiz (/)
Quando a página index.ts carrega

Quando useRedirect é montado

     E o conteúdo da página é exibido em japonês

Dado que o build é executado
Quando getStaticPaths resolve
Então três arquivos HTML são gerados: out/en/dev/.../index.html, out/ja/dev/.../index.html, out/pt-BR/dev/.../index.html
```
