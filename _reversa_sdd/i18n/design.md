## Interface

| Símbolo | Assinatura | Retorno | Observação |
|---------|-----------|---------|------------|
| `useRedirect` | `() => void` | `void` | Hook montado em `pages/index.ts`; efeito colateral: `router.push` |
| `languageDetector.detect` | `() => string` | `string` | Retorna locale detectado: `'en'` \| `'ja'` \| `'pt-BR'` |
| `serverSideTranslations` | `(locale, namespaces) => Props` | `object` | Chamado em `getStaticProps` — carrega strings em build time |
| `ChangeLanguage` | `()` | `JSX.Element` | Widget de seleção de idioma; sem props |
| `getStaticPaths` | `() => { paths, fallback }` | `object` | Gera 3 rotas em build time |

**`useRedirect` — implementação core:**

```js
// src/lib/redirect.js
function useRedirect() {
  const router = useRouter()

  useEffect(() => {               // ⚠️ sem array de deps — roda em todo render
    const { pathname } = router
    if (pathname === '/404') return   // guard: evita loop em 404

    const detectedLocale = languageDetector.detect()
    const target = `${prefix}/${detectedLocale}/dev/gabriel-toshinori-nakano/`

    if (router.asPath !== target) {
      languageDetector.cache(detectedLocale)
      router.push(target)
    }
  })                              // ← ausência de [] é o bug documentado
}
```

**Decisão do autor (Pergunta 7):** o `useEffect` sem deps foi intencional. Alternativa aprovada para a reimplementação: `useEffect([router.asPath])` — executa quando a rota muda, não em todo render. O guard `router.asPath !== target` permanece como segurança adicional. 🟢

## Fluxo Principal — Primeira Visita (Detecção Automática)

```
Visitante acessa gtoshinakano.github.io/fullstack-profile/
  ↓
GitHub Pages serve pages/index.ts (HTML estático)
  ↓
React hidrata — useRedirect é montado
  ↓
languageDetector.detect():
  1. Verifica cookie/localStorage de preferência salva
  2. Se não encontrado: lê navigator.languages[0]
  3. Normaliza para 'en' | 'ja' | 'pt-BR' (fallback: 'en')
  ↓
pathname === '/404'? → NÃO → prossegue
  ↓
router.asPath !== target? → SIM (estamos em '/', não em '/pt-BR/dev/...')
  ↓
languageDetector.cache('pt-BR')   ← salva preferência
router.push('/fullstack-profile/pt-BR/dev/gabriel-toshinori-nakano/')
  ↓
GitHub Pages serve pages/[locale]/dev/.../gabriel.tsx para pt-BR
  ↓
Página carrega com strings PT-BR via next-i18next
```

## Fluxo Principal — Build Time (getStaticPaths)

```
npm run build
  ↓
Next.js chama getStaticPaths() em gabriel.tsx
  ↓
Retorna paths:
  [
    { params: { locale: 'en' } },
    { params: { locale: 'ja' } },
    { params: { locale: 'pt-BR' } },
  ]
  fallback: false
  ↓
Para cada locale, getStaticProps é chamado:
  serverSideTranslations(locale, ['common'])
  → carrega public/locales/{locale}/common.json
  → serializa strings no props do componente
  ↓
Build gera:
  out/en/dev/gabriel-toshinori-nakano/index.html
  out/ja/dev/gabriel-toshinori-nakano/index.html
  out/pt-BR/dev/gabriel-toshinori-nakano/index.html
```

## Fluxo Principal — Troca Manual de Idioma

```
Visitante está em /en/dev/... e clica em "日本語" no ChangeLanguage widget
  ↓
ChangeLanguage chama:
  router.push(`${prefix}/ja/dev/gabriel-toshinori-nakano/`)
  // prefix = '/fullstack-profile' em produção, '' em dev
  ↓
Next.js navega client-side para o novo path
  ↓
Componente re-hidrata com i18nProps do locale 'ja' (já no bundle)
  ↓
useTranslation('common') retorna strings japonesas
```

## Fluxos Alternativos

- **Locale em cookie/localStorage:** `languageDetector` verifica cache antes de ler `navigator.languages`. Visitante que trocou para `ja` manualmente volta a `ja` na próxima visita
- **Rota `/404`:** `useRedirect` detecta `pathname === '/404'` e retorna imediatamente sem push — evita loop infinito
- **Locale não suportado no browser:** `languageDetector` faz fallback para `'en'` por não encontrar match em `['en', 'ja', 'pt-BR']`
- **Build sem locale em `getStaticPaths`:** `fallback: false` — qualquer rota não gerada retorna 404 (estático)

## Dependências

- **`next-language-detector`** — detecção de locale do browser (wrapper sobre i18next-browser-languageDetector)
- **`next-i18next`** — `serverSideTranslations`, `appWithTranslation` wrapper em `_app.tsx`
- **`react-i18next`** — `useTranslation` hook consumido pelas features
- **`next/router`** — `useRouter()` para `router.push` e `router.events`
- **`prefix.ts`** — basePath helper — crítico para `router.push` no ChangeLanguage (BR-09)
- **`public/locales/{en,ja,pt-BR}/common.json`** — arquivos de tradução (34+ chaves documentadas)
- **`next-i18next.config.js`** — `defaultLocale: 'en'`, `locales: ['en', 'ja', 'pt-BR']`

## Decisões de Design Identificadas

| Decisão | Evidência no código | Confiança |
|---------|---------------------|-----------|
| Client-side redirect em vez de built-in Next.js i18n | ADR-002 — commit `63139d7` "Removed serverside translation" | 🟢 |
| Route-based locale (`/[locale]/...`) em vez de subdomínio ou query param | `pages/[locale]/dev/...` estrutura de pastas | 🟢 |
| `serverSideTranslations` em build time (não runtime) | `getStaticProps` em `gabriel.tsx` + `output: 'export'` | 🟢 |
| `next-language-detector` para detecção + cache de preferência | `src/lib/languageDetector.js` | 🟢 |
| `prefix` obrigatório em todos os `router.push` de locale | `ChangeLanguage/index.tsx` — BR-09 + commit `e33d3e7` | 🟢 |
| `reloadOnPrerender: true` — configuração sem efeito (dead option) | `next-i18next.config.js:5` — ADR-002 | 🟢 |

## Estado Interno

Nenhum estado React nesta feature. Estado de locale é mantido por:
- **URL** — locale na rota (`/ja/dev/...`) é a fonte primária de verdade
- **Cookie/localStorage** — `languageDetector` persiste preferência do visitante entre sessões
- **`next-i18next` context** — strings carregadas via props do `getStaticProps` e distribuídas via Context do `appWithTranslation`

## Observabilidade

Nenhum evento GA específico para troca de idioma. O pageview automático captura indiretamente (a URL contém o locale).

## Riscos e Lacunas

- ~~🔴 useEffect sem deps~~ → **Resolvido:** implementar `useEffect([router.asPath])` na reimplementação — comportamento original preservado sem execução redundante (✅ Pergunta 7) 🟢
- 🟡 **Múltiplos hops de redirect:** primeira visita passa por `/` → `/{locale}/dev/...`. Em conexões lentas, o visitante pode ver uma tela em branco por um frame antes do redirect. Sem loading state nesta transição.
- 🟡 **`reloadOnPrerender: true`** pode ser removido de `next-i18next.config.js` sem impacto — mas a remoção deve ser documentada para evitar reintrodução por engano.
- 🟡 **Adicionar novo locale requer mudanças em 4+ arquivos** (ver `_reversa_sdd/traceability/spec-impact-matrix.md#Cenário A`) — sem automação ou validação de completude.
