## Visão Geral

A feature Layout encapsula a infraestrutura de shell da página: o componente `Public` que injeta `<head>`, fontes, scripts CDN (Pace.js, Unicons) e o widget `ChangeLanguage`; e os helpers `prefix.ts` e `customLoader.ts` que resolvem os desafios do basePath do GitHub Pages e da incompatibilidade do otimizador de imagens Next.js com static export. É a fundação sobre a qual todas as outras features são montadas.

## Responsabilidades

- Injetar metadados de `<head>` (title, description, viewport, OG tags) 🟢
- Carregar fontes Google Fonts (Syne) via `<link>` no head 🟢
- Injetar scripts CDN: Pace.js (loading indicator) e Unicons (icon font) 🟢
- Disponibilizar o widget `ChangeLanguage` na interface 🟢
- Fornecer o helper `prefix` para resolução de basePath em todos os assets 🟢
- Fornecer o `customLoader` para bypass do otimizador de imagens Next.js 🟢

## Regras de Negócio

- **ADR-001 / ADR-004** — `customLoader` é uma função identidade que retorna `src` sem modificação. É o mecanismo que torna `<Image>` compatível com `output: 'export'`. 🟢
- **BR-09** — `prefix` deve ser usado em **todos** os `router.push` de troca de locale e em **todas** as URLs de assets que precisam do basePath. Em desenvolvimento, `prefix` é string vazia. 🟢
- Pace.js será hospedado localmente em `public/pace.min.js` na reimplementação, eliminando a dependência de CDN externo. 🟢 [Decisão confirmada — Pergunta 8]
- Unicons é carregado via `<link>` CSS CDN — se o CDN cair, ícones são substituídos por espaços vazios. 🟡
