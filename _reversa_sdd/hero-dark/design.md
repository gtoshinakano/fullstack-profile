## Fluxo Principal — Troca de Aba

1. Visitante clica em botão de aba (ex: "Projects")
2. `onSelect('projects')` é chamado — `loading` vai a `true` (botões desabilitados)
3. GSAP anima saída dos elementos da aba atual: `stagger: 0.05 × n`, fade out + slide
4. GSAP `scrollTo: '#profile-photo'` rola o viewport para o elemento de foto do perfil 🟢
5. `selected` é atualizado para `'projects'`
6. GSAP anima entrada dos elementos da nova aba: fade in + slide
7. `loading` vai a `false` (botões reabilitados)
8. `ga.event({ action: 'menu_click', params: { event_category: 'projects' } })` é chamado 🟢

## Fluxo Principal — Accordion de Projetos

```
toggleByLabel(label):
  if expanded.includes(label):
    setExpanded(expanded.filter(l => l !== label))  // colapsa
  else:
    setExpanded([...expanded, label])                // expande

toggleAll():
  if allExpanded (expanded.length === data.length):
    setExpanded([])                                  // colapsa todos
  else:
    setExpanded(data.map(p => p.label))              // expande todos
```

⚠️ **Bug BR-07:** `expanded.includes(label)` testa igualdade de string. Com 5 projetos `label="requirement"`, incluir qualquer um inclui todos os 5 na mesma posição lógica — todos se expandem e colapsam juntos.

## Fluxo Principal — Carrossel de Fotos (ScrollTrigger)

1. `useLayoutEffect` em `HeroDark/index.tsx` registra GSAP ScrollTrigger no elemento `#profile-photo`
2. Configuração: `scrub:1`, `start:'top center'`, `end:'10% 10%'`
3. Timeline:
   - `photo-01` (`gabriel-photo.png`) — visível no início
   - Ao scroll: `photo-01` faz fade out enquanto `photo-02` (`gabriel-github.png`) faz fade in
   - Ao continuar o scroll: `photo-02` faz fade out enquanto `photo-03` (`gabriel-photo.jpg`) faz fade in
4. Scroll para cima reverte a sequência (scrub bidirecional)

## Fluxo Principal — Menu Sticky

1. ScrollTrigger monitora entrada da seção `#hero-dark` no viewport
2. Ao entrar: classes CSS são adicionadas ao menu lateral, expandindo sua largura/visibilidade
3. Implementado via `toggleClass` do GSAP ScrollTrigger ou via `onEnter`/`onLeave` callbacks
4. Não há estado React — puramente CSS-driven via GSAP class toggle 🟡

## Fluxos Alternativos

- **Clicar na aba já selecionada:** `onSelect` verifica `selected === tab` e retorna sem ação (sem animação, sem GA event) 🟡
- **Jobs sem educação:** `hasEducation=false` remove os dois registros JSX hardcoded do DOM via renderização condicional
- **Todos os projetos expandidos + clicar "Expand All":** `allExpanded=true` → `toggleAll()` colapsa tudo
- **Projetos com `label="requirement"`:** 5 projetos são expandidos/colapsados em bloco (ver BR-07)

## Riscos e Lacunas

- 🔴 **Bug BR-07 (label duplicado):** 5 projetos com `label:"requirement"` se expandem em bloco. Corrigir exige atualizar `toshi-projects.json` com labels únicos E migrar o state de `expanded[]` para usar `id` em vez de `label`.
- 🟡 **Estado do accordion não persiste:** ao trocar de aba (ex: ir para Jobs e voltar para Projects), `expanded` reseta para `[]`. Pode surpreender o visitante.
- 🟡 **`selected` não reflete na URL:** não há query string ou hash para compartilhar uma aba específica diretamente.
- 🔴 **Ausência de atributos ARIA:** botões de aba não têm `role="tab"`, `aria-selected`, `aria-controls` — impacto em leitores de tela.
- 🟡 **ScrollTrigger cleanup:** sem evidência de `ScrollTrigger.kill()` no unmount do componente.
